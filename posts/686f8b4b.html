5.1 树的基本概念

5.1.1 树的定义

树是一种 逻辑结构

树是 \(n(n\geq0)\)个结点的有限集合 \(,n=0\) 时, 称为 空树。

而任意 非空树 应满足:

  1. 有且仅有一个特定的称为 根 的结点。
  2. 当 \(n>1\) 时, 其余结点可分为 \(m(m>0)\) 个互不相交的有限集合, 其中每一个集合本身又是一棵树, 称为根结点的 子树。

树是一种递归的数据结构

树的特点

  1. 树的根结点没有前驱, 除根结点以外的所有结点有且只有一个前驱
  2. 树中所有结点可以有另个或多个后继

5.1.2 基本术语

  • 结点间的关系

    • 祖先结点和子孙结点
    • 双亲结点和孩子结点
    • 兄弟结点

  • 度: 树中的一个 子结点 的个数称为该结点的 度

    树中最大度数称为 树的度

  • 分支结点和叶子结点

    • 度大于 0 的结点称为 分支结点
    • 度为 0 的结点称为 叶子结点

  • 结点的深度、高度和层次

    • 结点的层次
    • 结点的高度: 从根结点开始自顶向下逐层累加
    • 结点的深度: 从叶结点开始自底向上逐层累加

  • 有序树与无序树

  • 路径

    路径 树中两个结点之间的路径是由这两个结点之间所经过的 结点序列 构成的

    树中的分支是有向的, 即从双亲结点指向孩子结点, 所以路径一定是 自上而下 的

    路径长度 路径上所经历 边 的个数

  • 森林

    森林 \(m(m\geq0)\) 棵互不相关的树的集合

5.1.3 树的性质

树具有以下最基本的性质

  1. 树中的结点数等于所有结点的度数 \(+1\) (根结点)。

  2. 度为 \(m\) 的树中第 \(i\) 层上至多有 \(m^{i-1}\) 个结点 \((i\geq1)\)。

  3. 高度为 \(h\) 的 \(m\) 叉树至多有 \((m^h-1)/(m-1)\) 个结点 (上一条结论求和)。

  4. 具有 \(n\) 个结点的 \(m\) 叉树的最小高度为:

    \((m^h-1)/(m-1)=n\quad\Longrightarrow\quad\lceil \log_m(n(m-1)+1)\rceil\)。

5.2 二叉树

5.2.1 二叉树的概念

1. 二叉树的定义

二叉树是 \(n(n\geq0)\) 个结点的有限集合 -- 逻辑结构

  1. \(n=0\) 时, 二叉树为空;
  2. \(n>0\) 时, 由根结点的两个互不相交的被称为根的左子树和右子树组成。左子树和右子树也分别是一棵二叉树。
五种基本形态

二叉树 vs 度为2有序树
  1. 二叉树可以为空, 而度为2的有序树至少有三个结点
  2. 二叉树的孩子结点始终有左右之分, 而度为2有序树的孩子结点次序是 相对的

2. 特殊二叉树

1. 满二叉树

一棵高度为 \(h\), 且含有 \(2^h-1\) 个结点的二叉树为满二叉树。

2. 完全二叉树

设一个高度为 \(h\)、有 \(n\) 个结点的二叉树, 当且仅当其每个结点都与高度为 \(h\) 的满二叉树中编号 1~n 的结点一一对应时, 称为完全二叉树。

性质:

  1. 若 \(i\leq\lfloor n/2\rfloor\), 则结点 \(i\) 为分支结点, 否则为叶子结点
  2. 叶子结点只可能在层次最大的两层上出现。对于最大层次的叶子结点, 都依次排在最左边的位置上。
  3. 度为1的结点若存在, 则可能有一个, 且是编号最大的分支结点, 并且孩子结点一定是 左结点。(重要特征)
  4. 若 \(n\) 为奇数, 则每个分支结点都有左节点和右节点; 若 \(n\) 为偶数, 则编号最大的分支结点只有左节点没有右节点
3. 二叉排序树

一棵二叉树, 若树非空则具有如下性质: 对任意结点若存在左子树或右子树, 则其左子树上所有结点的关键字均小于该结点, 右子树上所有结点的关键字均大于该结点。

4. 平衡二叉树

树上 任意节点 的左子树和右子树的深度只差不超过1。

3. 二叉树的性质

  1. 非定二叉树上的叶子结点数等于度为2的结点数加1, 即 \(n_0=n_2+1\)

    \(n_i\) 代表度为 \(i\) 的结点的个数

    \[\begin{cases} n=n_0+n_1+n_2\\ n=n_1+2n_2+1(根节点) \end{cases}\quad\Longrightarrow\quad n_0=n_2+1\]

  2. 非空二叉树上第 \(k\) 层上至多有 \(2^{k-1}\) 个结点 \((k>1)\)

  1. 高度为 \(h\) 的二叉树至多有 \(2^h-1\) 个结点 \((h>1)\)

  2. 对完全二叉树按从上到下、从左到有的顺序依次编号 \(1, 2, ..., n\), 则有以下关系:

  • 当 \(i>1\) 时, 结点的双亲结点标号为 \(\lfloor i/2\rfloor\), 即当 \(i\) 为偶数时, 其双亲结点的编号为 \(i/2\), 他是双亲结点的左孩子;当 \(i\) 为奇数时, 其双亲结点的编号为 \((i-1)/2\), 他是双亲结点的右孩子。

  • 当 \(2i\leq n\) 时, 结点 \(i\) 的左孩子编号为 \(2i\), 否则无左孩子。

  • 当 \(2i+1\leq n\) 时, 结点的右孩子编号为 \(2i+1\), 否则无右孩子。

  • 结点所在层次为 \(\lfloor \log_2i\rfloor+1\)。

    第 \(h\) 层: \(2^{h-1}\leq i<2^h\quad\Longrightarrow\quad h-1\leq\log_2i<h\)

  1. 具有 \(n\) 个 \((n>0)\) 结点的完全二叉树的高度为 \(\lfloor\log_2n\rfloor+1\) 或 \(\lceil\log_2(n+1)\rceil\)。

    1. 高度为 \(h\) 的二叉树至多有 \(2^h-1\) 个结点 \((h>1)\)

    \(h-1<\log_2(n+1)\leq h\quad\Longrightarrow\quad\lceil\log_2(n+1)\rceil\)

5.2.2 二叉树的存储结构

1. 顺序存储

用一组连续的存储单元依次自上而下、自左至右存储 完全二叉树 上的结点元素

  • 在 完全二叉树 中依次编号, 对于结点 \(i\) : 若存在左孩子, 则编号为 \(2i\); 若存在右孩子, 则编号为 \(2i+1\) -- (从数组下标1开始存储, 0存储结点个数)

    若从0开始存, 左孩子编号为 \(2i+1\), 右孩子编号为 \(2i+2\)

  • 在 非完全二叉树 中, 添加一个不存在的空节点, 在数组中用0表示

2. 链式存储

用链表来存放一棵二叉树, 二叉树中每个结点用链表的一个链结点来存储。

5.5 树与二叉树的应用

5.5.1 二叉排序树

1. 定义

二叉排序树 (BST, Binary Search Tree), 也称二叉查找树

二叉排序树可以为空树, 或者为非空树, 当为非空树时有如下特点:

  1. 若左子树非空, 则左子树上所有结点关键字值均根结点的关键字。
  2. 若右子树非空, 则右子树上所有结点关键字值均根结点的关键字。
  3. 左、右子树本身也分别是一棵二叉排序树。

默认没有相同结点

中序遍历序列:1, 2, 3, 4, 5, 7, 9, 10, 16

左子树 < 根 < 右子树 左子树结点值 < 根结点值 < 右子树结点值

2. 查找

  • 二叉树非空时, 查找根结点, 若相等则查找成功;
  • 若不等, 则当小于根结点值时, 查找左子树; 当大于根结点的值时, 查找右子树。
  • 当查找到叶节点仍没查找到相应的值, 则查找失败。

时间复杂度O(h) -- h 为二叉排序树的高度

3. 插入

  • 若二叉排序树为空, 则直接插入结点;
  • 若二叉排序树非空, 当值小于根结点时, 插入左子树; 当值大于根结点时, 插入右子树; 当值等于根结点时不进行插入。

4. 构造二叉排序树

  • 读入一个元素并建立结点, 若二叉树为空将其作为根结点;
  • 若二叉排序树非空, 当值小于根结点时, 插入左子树; 当值大于根结点时, 插入右子树; 当 值等于根结点时不进行插入。

值相同, 但是大小顺序不同, 构造出的二叉排序树也可能不同

5. 删除

过程比较复杂

  1. 若被删除的结点z是 叶子结点, 则直接删除。(删除1, 3, 7, 10)
  2. 若被删除的结点z只有 一棵子树, 则让z的子树称为父结点的子树, 代替z结点。(删除5, 16)
  3. 若被删除的结点z有 两棵子树, 则让z的中序序列直接后继代替z, 并删除直接后继结点。

在二叉排序树中删除并插入某节点, 得到的二叉排序树是否与原来相同

可能相同也可能不同, 如果删除和插入的是叶子结点, 则没有变化 (如删除7); 如果删除和插入的不是叶子结点, 就会发生改变 (如删除5)

6. 查找效率

后面详细介绍, 此处简单分析

平均查找长度 (ASL) 取决于树的高度。

例: {2, 1, 4, 3} {1, 2, 3, 4}

\(ASL=(1+2*2+3)/4=2\) -- \(ASL=(1+2+3+4)/4=2.5\) --

5.5.2 平衡二叉树

平衡二叉树(Balanced Binary Tree): AVL, 任意节点的 平衡因子 的绝对值不超过一。

平衡因子: 左子树高度-右子树高度

最小结点数

高度为h的最小平衡二叉树的结点数 \(N_h\) (最少)

\(N_h=N_{h-1}+N_{h-2}+1\) \(N_0=0\quad N_1=1\)

可以推算出高度为 h 的最小平衡二叉树的结点数

平衡二叉树的判断

利用递归的后序遍历过程:

  1. 判断左子树是一棵平衡二又树
  2. 判断右子树是一棵平衡二又树

1. LL平衡旋转(右单旋转)

原因: 在结点A的左孩子的左子树上插入了新结点

调整方法: 右旋操作 -- 将A的左孩子B代替A, 将A结点称为B的右子树根结点, 而B的原有子树则作为A的左子树。

2. RR平衡旋转(左单旋转)

原因: 在结点A的右孩子的右子树上插入了新结点 调整方法: 左旋操作 -- 将A的右孩子B代替A, 将A结点称为B的左子树根结点, 而B的原有子树则作为A的右子树。

3. LR平衡旋转(先左后右双旋转)

原因: 在结点A的左孩子的右子树上插入的新结点

调整方法 先左旋后右旋操作: 将A的左孩子B的右孩子结点C代替B, 然后再将C结点向上代替A的位置。

原因 在结点A的右孩子的左子树上插入了新结点

调整方法 先右旋后左旋操作: 将A的右孩子B的左孩子结点C代替B, 然后再将C结点向上代替A的位置。

5.5.3 哈夫曼树(赫夫曼树)

哈夫曼树

带权路径长度

路径长度: 路径上所经历边的个数 结点的权: 结点被赋予的数值

树的带权路径长度: WPL, 树中所有 叶结点 的带权路径长度之和, 记为:

哈夫曼树 : 也称 最优二叉树, 含有带 n 个带权叶子结点带权路径长度最小的二叉树

上例子中, 第二个树为哈夫曼树

哈夫曼树的构造算法
  1. 将 n 个结点作为 n 棵仅含有一个根结点的二叉树, 构成森林 F;
  2. 生成一个新结点, 并从 F 中找出根结点权值最小的两棵树作为它的左右子树, 且新节点的权值为两棵子树根结点的权值之和;
  3. 从 F 中删除这两棵树, 并将生成的树加入到 F 中;
  4. 重复2,3步骤, 直到树 F 中中有一棵树为止。

哈夫曼树的性质
  1. 每个初始结点都会成为叶结点, 双支节点都为新生成的结点
  2. 权值越大离根结点越近, 反之权值越小离根结点越远
  3. 哈夫曼树中,没有结点的度为1
  4. n 个子结点的哈夫曼数的结点总数为 \(2n-1\), 其中度为 2 的结点 \(n-1\)
